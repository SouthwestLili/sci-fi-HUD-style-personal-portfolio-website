2090.03.04 DEVLOG Raft Leader Election — Week 3 Notes ▶
Spent most of this week debugging a split-brain scenario in my Raft implementation. The issue was in how I handled heartbeat timeouts — nodes were timing out too close together, causing simultaneous elections with no winner. Fixed by adding randomized jitter (150–300ms) to election timeouts. Also discovered I was persisting log entries incorrectly before responding to AppendEntries — fixed the order. Network partition tests now pass consistently. Next up: snapshotting with InstallSnapshot RPC.
2090.02.25 RESEARCH Reading: Flash Attention Paper — Notes ▶
Read through the Flash Attention 2 paper (Dao 2023). Key insight: standard attention is memory-bandwidth bound, not compute-bound. FA reorders operations to minimize HBM reads/writes using tiling. The online softmax trick (stable incremental softmax) is elegant. Block sizes are chosen to fit in SRAM. FA2 improves parallelism across sequence dimension vs FA1. Practical takeaway: critical to fuse softmax + matmul kernels if writing custom CUDA. Plan to implement a simplified version as a learning exercise.

2090.02.11 DEVLOG Hobby Kernel: Setting Up Paging ▶
Finally got identity mapping working in the kernel project. Spent 6 hours tracking a page fault that turned out to be a wrong base address in the PML4 table. The x86-64 paging structure (PML4 → PDPT → PD → PT) is intimidating at first but makes sense once you map it out. Using limine bootloader to set up initial page tables; then taking over from there in C. Next milestone: virtual memory allocator so I can start placing kernel heap properly.
2090.02.04 RESEARCH Exploring Differential Privacy for ML ▶
Started reading into DP-SGD (Differentially Private SGD). The core idea: clip per-sample gradients to bound sensitivity, then add calibrated Gaussian noise before aggregating. Privacy budget ε tracks over training steps via the moments accountant. The privacy-utility tradeoff is steep at small ε — need large datasets to recover accuracy. Interesting open question: can we do better with structured noise or gradient compression? Going to try running DP-SGD on MNIST with opacus this weekend.
